# governance

single-maintainer project: [@sunnypatell](https://github.com/sunnypatell) owns direction, review, releases, and security response. no committee, no voting - honest about the bus factor of 1.

what that means in practice:

- **decisions**: made by the maintainer, in the open (issues/PRs), with reasons written down
- **contributions**: welcome via PR ([CONTRIBUTING.md](CONTRIBUTING.md)); contributor work is credited by handle in the changelog and release notes
- **releases**: cut by the maintainer following [RELEASING.md](RELEASING.md); every artifact carries verifiable provenance ([SECURITY.md](SECURITY.md#release-integrity))
- **security**: private disclosure per [SECURITY.md](SECURITY.md), acknowledged within 3 business days
- **continuity**: if the project is abandoned, the custom license still permits personal use of existing releases, and the build pipeline is fully documented in-repo so a fork can reproduce it
